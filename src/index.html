• Search

    <input type="search">

    Search acts the same as a text box but it is aestheticlly different. User is provided wiht an "X" to delete the text within the box

  • Tel

    <input type="tel">

    Tel is a mobile first desigin. On a mobile the keyboard changes from the normal QWERTY keyboard to a number pad.

  • URL

    <input type="url">

    URL is a mobile first design. On a mobile the keybaord is changed for easier user input. The space is replaced with commonly use keys for a URL.

  • Email

    <input type="email">

    Email is a mobile first design. On a mobile the keybaord is changed for easier user input. The spcae key is reduced to fit a @ and . key next to it.

  • Date

    <input type="date">

    Date allows a user to input a day, month and year - aka a date. The user is provided with a graphical calendar to input the date.

  • Week

    <input type="week">

    Week lets the user input a week. They are provided with the same calendar input as with date.

  • Month

    <input type="month">

    Month let the user input a month. They are provided with the same calendar input as with date.

  • Time

    <input type="time">

    Time lets the user input a time.

  • DateTime

    Not supported on any browser!

  • DateTime Local

    <input type="datetime-local">

    DateTimeLocal let the user iput a date and time with no timezone. They are provided with the same calendar input as with date.

  • Number

    <input type="number">

    Number let the user input a numebr.

  • Range

    <input type="range">

    Range provides the user with a sliding bar for input. The browser renders the input as a number.

  • Color

    <input type="color">

    Lets the user input a color. They are provided with a graphical color picker. The browser renders the input as a HEX value.
